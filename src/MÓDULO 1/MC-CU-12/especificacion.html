1. Descripción

El caso de uso MC-CU-12 permite al Administrador seleccionar un folio con retraso > 5 días y generar un aviso digital con justificación y prioridad, enviando notificación al área responsable mediante el sistema de mensajería interna. Se registra auditoría con fecha/hora y usuario, y se confirma la recepción.

2. Actores

  • Administrador: Inicia la solicitud, define prioridad y justificación.
  • Mensajería Interna: Recibe el aviso y confirma recepción.

3. Reglas de negocio

  • Solo perfiles con rol Administrador pueden enviar avisos.
  • El folio debe tener más de 5 días de retraso.
  • La justificación debe tener al menos 10 caracteres.
  • Se registra auditoría para alta de aviso y confirmación de recepción.

4. Flujo principal

  1. Administrador visualiza folios con retraso y selecciona uno.
  2. Captura justificación y prioridad.
  3. El sistema valida permisos y regla de retraso > 5 días.
  4. Se genera el aviso digital y se notifica a Mensajería Interna.
  5. Se registra auditoría y se confirma recepción (automática o manual).

5. Datos

Ver Scripts SQL para tablas folios, alertas y auditoría; datos de prueba realistas incluidos.

6. Diseño

  • Arquitectura MVC en front-end: Model (localStorage), View (DOM), Controller (eventos/reglas).
  • UI responsiva consistente con prototipos existentes.
  • Manejo de errores con mensajes descriptivos en español.
